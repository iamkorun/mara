Why mara?

The missing "find and report" step before running BFG or git filter-repo.

🔍
Blob-level detail
Unlike git-sizer aggregate stats, mara names the exact files — path, size, SHA, date, author.
⚡
Zero config
Run it in any git repo. No config files, no flags required, no setup. Just mara scan.
🦀
Single Rust binary
No runtime deps, no Java, no Python. cargo install mara and you're done.
🧹
Cleanup ready
mara suggest spits out a ready-to-run git filter-repo command with shell-escaped paths.

Under the hood

mara shells out to two git plumbing commands and does the heavy lifting for you:

# enumerate every object, filter blobs by size
git rev-list --objects --all | git cat-file --batch-check

# find the first commit that introduced each path
git log --all --diff-filter=A -- <path>

Blobs are deduplicated by OID so the same large file across multiple commits counts once. Results sorted descending by size. Dependencies: clap + anyhow — nothing else.
