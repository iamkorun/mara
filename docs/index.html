Why mara?

The missing "find and report" step before running BFG or git filter-repo.

🔍
Blob-level detail
Unlike git-sizer's aggregate stats, mara names the exact files — path, size, SHA, date, author.
⚡
Zero config
Run it in any git repo. No config files, no flags required, no setup. Just mara scan.
🦀
Single Rust binary
No runtime deps, no Java, no Python. cargo install mara and you're done.
🛠️
Cleanup ready
mara suggest emits ready-to-run git filter-repo commands, with shell-escaped paths.

How it works

mara shells out to two git plumbing commands and does the heavy lifting for you:

# enumerate every object, filter blobs by size
git rev-list --objects --all | git cat-file --batch-check

# find the first commit that introduced each blob
git log --diff-filter=A --find-object=<oid>

Blobs are deduplicated by OID so the same large file across multiple commits counts once. Results are sorted descending by size. Dependencies: clap and anyhow — nothing else.

Install

Requires Rust 1.70+ and git on your PATH.

cargo install mara
